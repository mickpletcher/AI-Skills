# spec-writer

A Cursor skill for turning rough repo work into numbered specs before coding begins.

## Source of Truth

Use [instructions.md](C:/Users/mick0/OneDrive/Documents/Code%20&%20Dev/GitHub/AI-Skills/cursor/skills/spec-writer/instructions.md) as the editable source of truth for this skill.

## How To Implement In Cursor

### Option 1: Project Rules

1. Open the target repo in Cursor.
2. Open the project rules surface.
3. Copy the full contents of [instructions.md](C:/Users/mick0/OneDrive/Documents/Code%20&%20Dev/GitHub/AI-Skills/cursor/skills/spec-writer/instructions.md).
4. Paste the content into the project rule.
5. Save it.
6. Test it against a real repo change that needs planning.

### Option 2: One Time Chat Context

1. Open Cursor Chat.
2. Paste the instruction content first.
3. Then provide the repo context and the change idea.
4. Ask Cursor to create the next numbered spec.

## First Test

```text
Use spec-writer to create the next numbered spec for this repo change, including requirements, plan, and tasks.
```
